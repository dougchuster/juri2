# Barra de Qualidade de Componentes

Um componente "pronto" deve:
- Ter API simples e previsivel (props coerentes e nomes consistentes).
- Suportar estados: hover/focus/active/disabled/loading.
- Ser acessivel: label/aria quando necessario, foco, teclado, role correto.
- Ser tematizavel via tokens (sem hardcode de cores/espacos).
- Ser responsivo por padrao.
- Ter historias/exemplos (Storybook ou pagina de componentes) quando existir infra.
- Ter testes minimos quando o projeto usar testes (ao menos smoke).
